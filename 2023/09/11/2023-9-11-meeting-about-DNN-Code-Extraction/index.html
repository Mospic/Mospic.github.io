模型定义提取方法和问题（目前存在）

提取方法：

目前使用的方法是，遍历Repo中的所有.py文件，对于所有在类声明中继承了nn.module和torch.nn.module以及声明了from torch.nn import Module as xxx 且类中继承了xxx的class源码进行了提取，对于每个Repo中所有的符合要求的class类名与源码放入一个List中，最后每个Repo都会有一个键值对RepoName:Extracted_Code_List，所有键值对组合成一个大的字典json.load进文件中。

  • 编码格式问题：大部分代码源文件的编码格式都可以通过chardet库来分辨，但是有一部分以Windows-开头（如Windows-1252）的会无法分辨（chardet置信为0），统一使用utf-8来编码，测试了几个Windows-开头的小众国家都可以分辨。

  • 提取关键字问题：目前提取的是继承了nn.module和torch.nn.module以及声明了from torch.nn import Module as xxx 且类中继承了xxx的class源码，后面可以对nn.module和torch.nn.module考虑正则匹配声明？，以及存在一些比较特殊的情况，比如：

    • import pytorch_lightning as pl
class ClassificationModel(pl.LightningModule):
    def __init__():
    def forward(self, x):

  • 树形问题：（目前还没做），可能考虑将List变换为树形，类似LeetCode那种List改Tree

  • 还需要提取的源码做什么东西？

    • 把实例的关系搞出来
